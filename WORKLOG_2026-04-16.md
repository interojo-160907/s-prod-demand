# 작업 로그 (2026-04-16)

## 목표
- `s관 부족수량.xlsx`를 기반으로 납기 기준 제품군(품명 + POWER) 단위로 공정별 부족을 집계하고, Streamlit 대시보드로 확인 가능하게 구성.

## 현재 결론(중요)
- **대시보드 공정별 부족 값은 `이니셜별` 시트의 공정 컬럼([10]/[20]/[45]/[55]/[80])을 그대로 집계**해서 보여주도록 변경함.
- 이전에 화면에서 0으로 보이던 케이스는 "재고 기반 계산(완료/가용)" 로직을 사용했기 때문이었고, 현재는 해당 방식 대신 공정 컬럼 기반 집계를 사용함.

## 주요 산출물
- 분석 스크립트: `excel_analysis.py`
- Streamlit 앱: `streamlit_app.py`
- 실행 스크립트: `run_app.ps1`
- Streamlit 설정: `.streamlit/config.toml`

## 생성되는 파일(out/)
- `out/납기_제품군_공정별부족.csv`
  - 컬럼: `제품군`, `납기일`, `필요수량`, `사출`, `분리`, `하이드레이션`, `접착`, `누수규격`
  - 의미: `이니셜별` 시트의 공정별 값 합계(납기일+제품군 기준)
- `out/제품코드_매핑누락.csv`
  - 설비별 시트 기반 `제품코드 -> 제품군` 매핑이 실패한 코드 목록(검토용)
- (참고) `out/납기_제품군_부족분석.csv`
  - 재고 기반(WIP/완료) 분석 버전(현재 대시보드에서는 사용하지 않음)

## 대시보드 화면(현재)
- 요약: `총 필요수량`만 표시
- 납기별 상세: `품명`, `POWER`, `납기일`, `사출`, `분리`, `하이드레이션`, `접착`, `누수규격`
- 필터/리스크/병목/총부족수량/괄호(WIP) 표기: **전부 제거**

## 주의/이슈
- 엑셀 파일에 `총합계` 행이 포함되어 있어, 데이터로 읽으면 합계가 2배로 커질 수 있음.
  - 현재 분석에서는 `총합계`/`종합계` 토큰을 감지해서 제외하도록 처리됨.
- Streamlit 캐시 때문에 CSV 갱신이 즉시 반영되지 않는 문제가 있었음.
  - 현재는 `mtime`(파일 수정시간)을 캐시 키로 사용하고, `분석 실행` 시 캐시를 clear 하도록 수정됨.
