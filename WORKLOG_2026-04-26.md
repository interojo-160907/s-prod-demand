# WORKLOG (2026-04-26)

## 사출 스케줄 개선 작업 로그

### 목표
- 납기일 최우선 유지
- 설비/라인구분/어레인지 매칭 기반 제품 배정 유지
- 같은 제품 내 POWER(도수) 배정에서 코아(POWER) 교체 최소화 및 “근접 POWER” 흐름 개선
- 현장 확인이 쉬운 상세표 제공(슬롯별 PW 표시)

### 변경사항 요약
- 스케줄 기간/시작일
  - `사출 계획` 기간을 5일로 고정, 시작일 입력 제거(항상 오늘 `_today_kst()` 기준): `streamlit_app.py`

- POWER 배정 로직(코아 교체 최소화 + 납기일 최우선)
  - POWER 잔량을 단순 int가 아니라 `{qty, due}`로 관리하여 POWER별 납기 반영: `streamlit_app.py`
  - 블록(주간/야간) 내 POWER 종류 기본 최대 2개
  - **납기일(min due) 그룹이 최우선**이며, 이전 POWER 유지/근접 선택은 동일 납기 그룹 안에서만 허용
  - 납기 위험(urgent: due <= block_day) POWER는 예외적으로 포함 가능

- 상세표(UI) 개선 (간트/그리드는 기존 유지)
  - 상세표에 `PW1..PW8` 컬럼 추가(각 슬롯 POWER를 칸별로 표시)
  - 상세표에서 `설비(상세)` 선택 추가(설비별 디테일링)
  - 간트/그리드는 제품 코드 표시 방식 유지(POWER/CHG 표기 제거)

- 엑셀 출력
  - `엑셀 다운로드 (운영양식)` 추가: 설비 x (일자/주야) 별로 `제품정보 / CAV / 도수 / 필요수량` 형태
  - `엑셀 다운로드 (캐비티별)` 추가: `날짜, 주야, 설비명, CAV, 도수, 필요수량` 롱포맷

### 메모
- CAPA 기준은 기존 유지: 생산실적 기반 중앙값 → 전체 중앙값 → 기본 24,000ea/일/설비 폴백
  - 24,000ea/일이면 블록당 12,000ea, 8슬롯이면 슬롯당 1,500ea 기준으로 배정됨

## 추가 작업 (UX/성능 안정화)

### UI/UX
- `사출 계획` 상단 제품군/필요수량 표시를 클릭 가능한 pills(필터) 대신 정보용 chips로 변경(눌러보게 되는 UX 제거).
- 사출 스케줄 생성은 항상 전체 데이터 기준으로 고정(제품군 클릭으로 스케줄이 바뀌지 않도록 “표시/스케줄” 완전 분리).

### 운영중(엑셀 raw `생산 제품`) 표시 규칙
- `사출` 시트의 `생산 제품`(품명)을 어레인지 `제품명`과 매칭해 `현재제품코드(R####)`를 산출하여 간트/상세표 표시.
- 규칙(5일 고정 기간):
  - 해당 설비가 5일 내 스케줄 배정이 1칸이라도 있으면: 미래 유휴칸에는 현재생산 표시를 퍼뜨리지 않음(유휴/사유 공란).
  - 해당 설비가 5일 내 스케줄 배정이 완전 0이면: 현재생산 제품을 5일 전체에 채워 “계속 돌리는 설비”를 표현.
  - 표시 목적이며 수량/POWER/납기 같은 임의 배정 값은 넣지 않음.

### 성능
- 큰 테이블은 Styler 생략(렌더링 지연 감소).
- `엑셀 다운로드(상세표)` 생성(openpyxl)은 캐시하여 재생성 최소화.
- 캐시 정확도: “현재 주간/야간(now block)”이 캐시에 반영되도록 키에 포함(교대시간 이후 표시 갱신).

### POWER 배정(추가 안정화)
- 블록(주/야) 내에서 **새 POWER를 추가할 때 “미세 잔량(예: 169ea)”만 위해 코아 교체가 생기지 않도록** 최소 배정량 규칙을 추가.
  - 기본: 이전 POWER가 아닌 “새 POWER”는 해당 블록에서 `min_new_type_qty(기본=slot_qty=2,000ea)` 이상 생산 가능할 때만 추가.
  - 단, 납기 임박(due <= block_day)인 urgent POWER는 예외적으로 포함.

### 제품 전환(색상매칭) 안정화
- 동일 납기 그룹 내에서도 **설비가 새 제품으로 전환하면서 블록에서 생산 가능한 수량이 너무 작으면(기본 2,000ea 미만)** 전환을 막고 블록을 유휴로 둠.
  - 목적: 컬러 매칭/세팅 churn 최소화(“49ea 때문에 제품 바뀜” 방지).
  - 단, 납기 임박(urgent: due <= day)인 경우는 예외.
 - 주간(블록1)에서 제품이 소진된 뒤 야간(블록2) 제품 선택 시에도 “전일 제품(prefer)”을 참조해 위 규칙이 적용되도록 처리.
